Axiomas y proposiciones auxiliares

En esta pagina, y por conveniencia, se listan todos los axiomas y proposiciones auxiliares requeridos para formalizar los invariantes de representación y las funciones de abstracción. Previamente se presentan los renombres de los tipos usados.

Renombres de tipos
  • Node es tupla(child: arreglo[2] de puntero(Node), parent: puntero(Node), color: Color, value: Value)
  • Value es Maybe(value_type)
  • value_type es tupla(clave: Key, significado: Meaning)

El TAD Maybe( ) representa un tipo extendido con un valor nulo. En otras palabras, el TAD Maybe se puede usar para representar los valores de los nodos, donde el nodo cabecera no tiene valor y los nodos internos sí tienen valor. (Esto independientemente de si se implementa con herencia o con un puntero o de otra forma.) Tiene dos observadores:

  • nothing?(x): que indica si x tiene un valor nulo, y
  • data(x): que devuelve el valor de x, suponiendo que no es inválido.

La especificación de este TAD queda como ejercicio (no obligatorio).

Axiomas y proposiciones auxiliares

En esta sección se deben incluir todos los axiomas y proposiciones auxiliares que se usen para describir los invariantes de representación, las funciones de abstracción, las precondiciones y las postcondiciones.

esDiccionario?
Retorna true si la secuencia representa un diccionario

esDiccionario?: secu(tupla( , )) bool
esDiccionario?(s) sinRepetidos?(primeros(s))

primeros
Proyecta las primeras componentes de una secuencia de pares

primeros: secu(tupla( , )) secu( )
primeros(s) if vacia?(s) then <> else (prim(s)) primeros(fin(s)) fi

hastaElem

headerValido
El nodo header no tiene valor. Su padre es la raíz, de color negro, y sus hijos derecho e izquierdo son el mayor y el menor valor del arbol respectivamente.

headerValido: Node n bool
headerValido(n) [n.color == Header nothing?(n.value) n.parent.color == Black (n.parent = (n.child[0] == &header) (n.parent n.child[0] = llegarAMinimo(n.parent)) (n.parent = (n.child[1] == &header) (n.parent n.child[1] = llegarAMaximo(n.parent))]

llegarAMinimo
Devuelve un puntero al minimo nodo del subarbol cuya raiz es el nodo parametro.

llegarAMinimo: puntero(node) n puntero(node) {n }
llegarAMinimo(n) if n.child[0] == NULL then n else llegarAMinimo(n.child[0]) fi

llegarAMaximo
Devuelve un puntero al minimo nodo del subarbol cuya raiz es el nodo parametro.

llegarAMaximo: puntero(node) n puntero(node)
llegarAMinimo(n) if n.child[1] == NULL then n else llegarAMinimo(n.child[1]) fi

nodosInternosValidos
Para cada nodo interno, sus ramas descendientes tienen la misma cantidad de nodos negros, y y el subarbol que tiene como raiz ese nodo cumple el invariante de Arbol binario de busqueda. Si es rojo, su o sus hijos son negros.

( n: Node) nodoHijo(n, e.header) (n.child[0] = n.child[0]->color = Black) (n.child[1] = n.child[1]->color = Black) nodosNegros(n.child[0]) = nodosNegros(n.child[1]) arbolBinarioDeBusqueda(n))

Termina

Termina(n) long(inorder(n)) < k

arbolBinarioDeBusqueda
Todos los nodos de la rama izquierda de n tienen valor menor al de n y todos los de la derecha, mayor.

arbolBinarioDeBusqueda: puntero(Node) n bool
arbolBinarioDeBusqueda(n) n->child[0] BOTTOM (n->child[0]->value < n->value arbolBinarioDeBusqueda(n->child[0]))) n->child[0] (n->child[0]->value > n->value arbolBinarioDeBusqueda(n->child[0])))
