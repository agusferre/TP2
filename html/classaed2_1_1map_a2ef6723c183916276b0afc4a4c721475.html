template<class Key, class Meaning, class Compare = std::less<Key>>
iterator aed2::map< Key, Meaning, Compare >::insert_or_assign ( const_iterator  hint,
const value_type &  value 
) [inline]

Inserta o Color::Redefine value en el diccionario.

Inserta un valor en el diccionario. Si *this ya tiene un valor con clave value.first, entonces se Color::Redefine. El iterador de retorno apunta al elemento recien insertado o redefinido.

Parámetros:
valuevalor a insertar
hintiterador apuntando al diccionario. Se espera que apunte al mínimo valor mayor o igual a value.first (i.e., a lower_bound(value.first)). Igualmente, la función es robusta y funciona correctamente aunque esto no ocurra.
Valores devueltos:
resiterador apuntando al elemento insertado o redefinido
Aspectos de aliasing:
\1
Precondición:
P { *this d_{ 0} SecuSuby(hint) DiccASecu(d_{ 0})}
Postcondición:
{*this definir(key, *this) coleccion(res) coleccion(hint) HaySiguiente(res) Siguiente(res) value}
Complejidad Temporal
  • Peor caso: ( (size(*this)) cmp(*this) copy(value))
  • Si hint apunta al primer valor con clave al menos value.first (o this->end() en caso de no existir): (cmp(*this) copy(value)) amortizado.
Atención:
Para prevenir que el nuevo valor redefina uno ya existente, usar aed2::map::insert.
Comentarios:
Esta función es similar al operator[], pero tiene dos ventajas y una desventaja. Las ventajas es que 1. se puede indicar un hint para la búsqueda y 2. no es necesario que Meaning tenga constructor sin parámetros. La desventaja es que la notación no es tan bonita.

Busca el valor de manera similar al operator[] Si lo encuentra, modifica el significado del nodo encontrado. Sino inserta un nuevo valor llamando a insert_rapido, con la seguridad de que it esta correctamente ubicado.
